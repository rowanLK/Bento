bento

Bento module, main entry point to game modules and managers. Start the game by using Bento.setup(). After this you have access to all Bento managers:
• Bento.assets
• Bento.audio
• Bento.input
• Bento.object
• Bento.savestate
• Bento.screen

setup(settings)

Setup game. Initializes all Bento managers.
Parameters:
Name Type Description
settings Object settings for the game
Properties
Name Type Attributes Description
assetGroups Object <optional>
 Asset groups to load. Key: group name, value: path to json file. See module:bento/managers/asset#loadAssetGroups
renderer String Renderer to use. Defaults to "canvas2d". To use "pixi", include the pixi.js file manually. Make sure to download v3!.
canvasDimension Rectangle base resolution for the game. Tip: use a bento/autoresize rectangle.
manualResize Boolean Whether Bento should resize the canvas to fill automatically
sortMode Boolean Bento Object Manager sorts objects by their z value. See module:bento/managers/object#setSortMode
subPixel Boolean Disable rounding of pixels
pixelSize Number Defaults to 1. You may resize pixels by setting this value. A kind of cheating with pixelart games.
preventContextMenu Boolean Stops the context menu from appearing in browsers when using right click
reload Object Settings for module reloading, set the event names for Bento to listen
Properties
Name Type Description
simple String Event name for simple reload: reloads modules and resets current screen
assets String Event name for asset reload: reloads modules and all assets and resets current screen
jump String Event name for screen jump: asks user to jumps to a screen
dev Boolean Use dev mode (for now it's only used for deciding between using throws or console.log's). Optional, default is false.
screenshot Object Event name for taking screenshots
onComplete function Called when game is loaded
